로그인 페이지
- 로고 하나 정도 추가
- 아이디(이메일), 패스워드

회원가입 페이지
- 입력 받을 사항 : 아이디(이메일), 패스워드, 패스워드 확인, 병원명, 병원주소, 병원이메일, 병원전화번호
- 마케팅 수신 동의 체크 박스들이 필요하다.

헤더(네비게이션 바)
- 모든 페이지 상단에 위치
- 대시보드, 하드웨어 관리, 환자 관리, 기록 보기 페이지로 이동할 수 있는 버튼이 필요
- 로그인 여부 확인(로그인 시에는 로그아웃 버튼을 만든다.), 내정보 페이지로 갈 수 있게 한다.

대시보드 페이지(사실상 메인 페이지)
- 아래의 각 항목은 컴포넌트 단위로 나눠야 함.
- 디바이스 및 허브 현황 : 등록된 허브 수, 등록된 디바이스 수, 연결된 디바이스 수, 가용중인 디바이스 수
- 간이 모니터링 섹션 : 현재 연결중인 디바이스에서 측정한 현재의 심박수, 산포도, 온도, 배터리 데이터 표시
- 초기 사용시 사용 가이드나 사용 설명서를 볼 수 있는 큰 버튼

하드웨어 관리 페이지
- 허브 관리, 디바이스 관리로 나눠야 함
- 허브 관리에서는 usb 연결을 통해 허브에게 wifi id, pw, 현재 로그인된 user_email을 전달할 수 있다.
- 허브 관리에서는 허브를 삭제하거나 등록할 수 있고 허브명을 바꿀 수 있다.
- 디바이스 관리에서는 허브에서 연결을 감지한 디바이스를 확인할 수 있고 각 디바이스에 디바이스명을 바꿀 수 있다.
- 디바이스 관리에서는 연결한 디바이스 수를 확인할 수 있고 각 디바이스에 어떤 환자가 연결되어있는지 알 수 있고 변경할 수 있다.

환자 관리 페이지
- 입원한 환자들을 등록받아 리스트업 하는 화면이다.
- 환자등록 버튼이 있고 등록된 환자들의 정보를 확인할 수 있으며 수정도 가능하다.
- 해당 환자에 연결된 디바이스가 있다면 연결된 디바이스명도 보일 수 있으면 좋을 것 같다.
- 해당 환자에게 연결된 디바이스를 연결 해제하거나 다른 디바이스로 바꿀 수도 있어야 한다.
- 현재 입원된 환자와 퇴원조치된 환자를 구분하여 관리할 수 있다.(환자 리스트에서 퇴원 처리를 할 수 있게 한다.)

기록 관리 페이지
- 기록된 csv 파일을 다운로드 받을 수 있고 삭제할 수 있다.
- csv 파일은 날짜별로, 환자별로 볼 수 있어야 한다.
- csv 파일은 최신의 파일을 먼저 볼 수 있어야 한다.
- 다양한 sorting 기능이 들어가면 좋을 것 같다.
- sorting된 파일들을 모두 다운로드 받을 수 있는 기능도 있으면 좋을 것 같다.

내정보 페이지
- 회원가입시에 입력한 정보를 확인하거나 수정할 수 있다.
- 회원 탈퇴 기능을 넣는다.
- 마케팅 수신 동의와 관련된 설정을 할 수 있게 한다.
